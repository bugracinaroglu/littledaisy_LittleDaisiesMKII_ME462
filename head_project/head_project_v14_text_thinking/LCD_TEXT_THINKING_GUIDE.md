# LCD Text and THINKING Face Guide

## 1. Raspberry Pi Python API

The active control mode must match the command source.

```python
robot_head.show_oopsie_daisy(
    hold_ms=5000,
    source=ControlMode.MANUAL,
)

robot_head.show_text(
    "Hello from Daisy",
    hold_ms=4000,
    italic=False,
    source=ControlMode.MANUAL,
)

robot_head.show_thinking(
    hold_ms=4000,
    source=ControlMode.MANUAL,
)
```

For ROS callbacks, use `source=ControlMode.ROS`.

### Duration behavior

- `hold_ms=5000` displays the message for five seconds.
- `hold_ms=0` keeps it visible until an explicit face, gesture, or center
  command replaces it.
- While a text overlay is visible, IMU `RUNNING` and `DIZZY` displays do not
  replace the text.

### Text limitations

The RP2350 LCD uses the built-in MicroPython 8x8 ASCII font.

- New lines are converted to spaces.
- Text is limited to 80 characters.
- Long text is wrapped automatically.
- The italic appearance is generated by shearing the glyph pixels in software.
- Standard English ASCII is recommended. Turkish-specific glyphs may not be
  available in the built-in font.

## 2. MANUAL keyboard control

Start the Raspberry Pi program, click the OpenCV window, then press `2`.

```text
9  -> THINKING face
[  -> italic "Oopsie Daisy"
]  -> general text configured in robot_head_project/config.py
```

The general manual text settings are:

```python
MANUAL_TEXT_HOLD_MS = 5000
MANUAL_TEXT_MESSAGE = "Hello from Daisy"
MANUAL_TEXT_ITALIC = False
OOPSIE_DAISY_HOLD_MS = 5000
```

## 3. RP2350 serial protocol

```text
FACE:THINKING,4000
TEXT:5000,1,Oopsie Daisy
TEXT:4000,0,Hello from Daisy
```

Format:

```text
TEXT:hold_ms,italic,text
```

The final text field may contain commas.

## 4. ROS2 topics

First select ROS control mode:

```bash
ros2 topic pub --once /robot_head/control_mode   std_msgs/msg/String "{data: 'ROS'}"
```

Show the thinking face:

```bash
ros2 topic pub --once /robot_head/face   std_msgs/msg/String "{data: 'THINKING,4000'}"
```

Or use the dedicated empty topic:

```bash
ros2 topic pub --once /robot_head/thinking   std_msgs/msg/Empty "{}"
```

Display general text:

```bash
ros2 topic pub --once /robot_head/text   std_msgs/msg/String "{data: 'Hello from Daisy'}"
```

Advanced text format:

```bash
ros2 topic pub --once /robot_head/text   std_msgs/msg/String "{data: '5000|1|Oopsie Daisy'}"
```

The ROS topic format is:

```text
hold_ms|italic|text
```

Dedicated Oopsie Daisy topic:

```bash
ros2 topic pub --once /robot_head/oopsie_daisy   std_msgs/msg/Empty "{}"
```

## 5. ROS2 services

```bash
ros2 service call /robot_head/oopsie_daisy_service   std_srvs/srv/Trigger "{}"

ros2 service call /robot_head/thinking_service   std_srvs/srv/Trigger "{}"
```

General dynamic text uses `/robot_head/text`, because the standard Trigger
service has no text request field.
